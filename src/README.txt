Decripcion:
Se ha utilizado passport para poder ocultar la informacion en la URL y hacer mas segura la sesión
ademas de que las contraseñas han sido encriptadas al momento del registro del usuario.
la idea es de tener un directorio con un nombre unico encriptado para el usuario, ademas se planeo pero no se concreto
usar BSon para serializar documentos.

Requisitos de uso:
-----------------
nodejs, mongo DB

Para ejecutar en servidor seguimos los siguientes pasos:'
1.- Iniciar el servicio de Mongo con mongod
2.- Situarse en el directorio del proyecto y ejecutar 'npm install' para las dependecnias y 'node start'
    para poner en marcha la aplicacion.
3.- Ingresar en cualquier navegador a la dirección 'localhost:3000/'
